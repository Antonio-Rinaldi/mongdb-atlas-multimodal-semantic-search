Cos'è una Ricerca Semantica Ibrida?

  • Combina ricerca tradizionale e semantica
  • La ricerca tradizionale filtra i risultati su cui andrà ad operare la ricerca semantica
  • Tramite l'utilizzo di embedding vettoriali la ricerca semantica può comprendere il contesto ed il significato dell'input dell'utente

Struttura di un indice vettoriale ibrido

{
  "name": "<name_of_the_index>",
  "definition": {
    "fields": [
      {
        "type": "vector",
        "path": "<name_of_embedding_field>",
        "numDimensions": <number_of_dimension>,
        "similarity": "euclidean | cosine | dotProduct",
        "quantization": "none | scalar | binary"
      },
      {
        "type": "filter",
        "path": "<field_used_for_filter>"
      }
    ]
  }
}

Creazione Indice Vettoriale con Mongosh

mongosh "mongodb://localhost:27017/<name_of_database>" \
 --eval 'db.getCollection("commercial-activities").createSearchIndex(
  "embeddings_town_vector_index",
  "vectorSearch",
  {
    "fields": [
      {
        "type": "vector",
        "path": "embeddings",
        "numDimensions": 1024,
        "similarity": "cosine",
        "quantization": "none"
      },
      {
        "type": "filter",
        "path": "town"
      }
    ]
  }
)'

ANN vs ENN

  • ANN (Approximate Nearest Neighbors)
    • Ricerca approssimata
    • Performance migliori
  • ENN (Exact Nearest Neighbors)
    • Risultati esatti
    • Più costoso computazionalmente

Aggregation Pipeline per Ricerca Vettoriale

db.getCollection('commercial-activities').aggregate([
  {
    $vectorSearch: {
      index: "embeddings_town_vector_index",
      path: "embeddings",
      queryVector: [],
      numCandidates: 4096,
      limit: 100,
      filter: {
        $and: [{ town: "<town>" }]
      }
    }
  },
  {
    $project: {
      _id: 0,
      embeddings: 0,
      score: { $meta: "vectorSearchScore" }
    }
  }
])
